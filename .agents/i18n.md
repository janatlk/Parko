# i18n Agent

## Purpose
Manage translations for the Parko fleet management system across 3 languages: Russian (RU), English (EN), Kyrgyz (KY).

## Trigger
When the user says: "add translations...", "translate...", "missing i18n keys..."

## Workflow

1. **Identify new keys needed** — scan the code for hardcoded strings or `t('key')` calls without translations
2. **Add to `frontend/src/shared/i18n/index.ts`** in ALL 3 language sections:
   ```ts
   ru: {
     translation: {
       module: {
         new_key: 'Русский текст',
       }
     }
   },
   en: {
     translation: {
       module: {
         new_key: 'English text',
       }
     }
   },
   ky: {
     translation: {
       module: {
         new_key: 'Кыргызча текст',
       }
     }
   },
   ```
3. **Naming convention:** `module.page.element` (e.g., `reports.cost_per_km.total_distance`)
4. **Check for conflicts** — no duplicate keys
5. **Verify** — search for the key in code to ensure it's used

## Rules
- ALWAYS add all 3 languages (RU, EN, KY)
- Russian is the primary language — write RU first, then translate
- Keep Kyrgyz translations consistent with Russian meaning
- Use consistent terminology (e.g., always "автопарк" for fleet, not mix with "парк")
- Don't translate: brand names, technical terms (VIN, API, JSON)
- Check that no hardcoded strings remain in UI components
- When adding a new module, create the full i18n structure upfront
