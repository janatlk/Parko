# AI Assistant — Data Manipulation Plan

## Phase 1: Backend — Tool Calling System

1. Создать `ai/tools.py` — функции-инструменты для операций с данными:
   - `list_cars(company, filters)` — список машин
   - `add_car(company, user, data)` — добавить машину
   - `update_car(company, user, car_id, data)` — обновить машину
   - `delete_car(company, user, car_id)` — удалить машину
   - `add_fuel(company, user, car_id, data)` — добавить запись топлива
   - `add_spare(company, user, car_id, data)` — добавить запчасть
   - `add_insurance(company, user, car_id, data)` — добавить страховку
   - `add_inspection(company, user, car_id, data)` — добавить осмотр
   - `delete_record(company, user, model, record_id)` — удалить запись

2. Каждый инструмент:
   - Проверяет что запись принадлежит компании пользователя
   - Вызывает существующие сериализаторы для валидации
   - Возвращает результат или ошибку

3. Обновить `ai/services.py` — добавить систему tool calling:
   - Системный промпт обновить: AI может использовать инструменты для изменения данных
   - AI определяет намерение (создать/обновить/удалить/прочитать)
   - AI извлекает параметры из запроса (car_id, поля, значения)
   - AI вызывает нужный инструмент, получает результат, формулирует ответ

4. Обновить Groq API вызов — использовать function calling:
   - Определить tools в формате Groq function calling
   - Отправлять запрос с tools параметром
   - Обрабатывать response: если tool_calls — вызвать функцию, отправить результат обратно

## Phase 2: Backend — Safety Layer

5. Добавить систему подтверждений:
   - Для операций записи/удаления AI возвращает "предложение" а не выполняет сразу
   - Фронтенд показывает пользователю подтверждение
   - Пользователь подтверждает → AI выполняет действие

6. Логирование всех действий AI:
   - Сохранять в `AIChatMessage` какие инструменты вызывались
   - Кто, когда, что изменил

7. Permissions:
   - AI наследует права пользователя (IsCompanyAdmin для изменений)
   - Если пользователь не может удалить — AI тоже не может

## Phase 3: Frontend — Confirmation UI

8. Обновить `AIPage.tsx` — добавить отображение действий:
   - AI: "Я хочу добавить машину BMW X5 (A123BC). Подтвердить?"
   - Кнопки "Подтвердить" / "Отменить"
   - При подтверждении — второй запрос к AI с флагом execute

9. Добавить визуальные индикаторы:
   - Зелёная галочка — действие выполнено
   - Красный крестик — ошибка/отмена
   - Иконка редактирования для изменений данных

## Phase 4: Frontend — Intent Detection

10. Обновить ввод — AI понимает контекстные команды:
    - "Добавь машину Toyota Camry, номер А456ВС"
    - "Удали машину с номером А123ВС"
    - "Обнови пробег у А456ВС на 15000 км"
    - "Добавь 50 литров топлива для А456ВС за март"

11. AI возвращает структурированное действие:
    - `action`: create/update/delete/read
    - `model`: car/fuel/spare/insurance/inspection
    - `params`: {field: value, ...}
    - `confirmation_required`: true/false

## Phase 5: Testing

12. Протестировать все CRUD операции через AI
13. Протестировать безопасность (данные чужой компании недоступны)
14. Протестировать отказ на нерелевантные вопросы
