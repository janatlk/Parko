# AI Assistant Implementation Plan

## Phase 1: Backend Setup

1. Создать модель `AIChatMessage` (company, user, role, content, created_at)
2. Создать миграцию для модели
3. Создать сервис `ai_assistant.py` с системным промптом и логикой
4. Создать API endpoint `POST /api/v1/ai/chat/` — принимает вопрос, возвращает ответ
5. Системный промпт: ты — AI-помощник Parko, знаешь только данные компании, отказываешься отвечать на вопросы не про Parko
6. Реализовать сбор контекста — авто, топливо, ТО, страховки, осмотры текущей компании
7. Реализовать проверку релевантности — если вопрос не про Parko, вернуть отказ
8. Добавить поддержку openai/ollama через env переменную `AI_PROVIDER` и `AI_API_KEY`
9. Добавить permission `IsCompanyMember` к endpoint
10. Добавить сериализатор для запроса/ответа

## Phase 2: Frontend

11. Создать компонент `AIPage.tsx` — простой чат-интерфейс
12. Добавить роут `/ai` в `App.tsx`
13. Добавить ссылку "AI" в навбар `AppLayout.tsx`
14. Создать API функцию `sendAIMessage(question)` → `ai/chat/`
15. Создать хук `useAIMessage` для отправки/получения
16. Реализовать UI: поле ввода, кнопка отправки, список сообщений
17. Добавить переводы (ru, en, ky) для AI страницы
18. Добавить индикатор загрузки при ответе ИИ
19. Добавить обработку ошибок (нет API ключа, отказ, timeout)

## Phase 3: Configuration

20. Добавить `AI_PROVIDER`, `AI_API_KEY`, `AI_MODEL` в `.env.example`
21. Добавить AI ключ в настройки компании (на будущее)
22. Протестировать с mock-данными
