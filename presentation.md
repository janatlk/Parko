Структура презентации (10–12 слайдов)

1. Титульный слайд
- Название продукта / проекта
- Формат ВКР (цифровой сервис / игра / ML / MVP и т.д.)
- ФИО студента, группа
- руководитель ВКР

2. Идея и ценность продукта
- Ключевая идея (1–2 предложения)
- Ценность для пользователя

3. Проблема и актуальность
- Какая реальная проблема существует, Почему она актуальна сейчас (рынок, тренды, боль)

4. Целевая аудитория
- 3 ключевых типа пользователей(кейс портреты)
- Краткое описание (кто, зачем, контекст использования)

5. Анализ рынка и конкурентов
- Прямые конкуренты (3)
- Косвенные аналоги
- Чем продукт отличается
- Краткий вывод по рынку

6. SWOT-анализ
- Strengths - сильные стороны
- Weaknesses - слабые стороны
- Opportunities - возможности
- Threats - угрозы

Можно в виде таблицы

7. Основные функции и механики продукта
- Core-функции (3–5 пунктов)
- Ключевые механики / логика работы
- Для игр: игровой цикл, прогресс, экономика

8. Пользовательские сценарии
- User Flow / CJM (упрощённо)
- Точка входа → ключевые шаги → результат
- Критические точки взаимодействия

9. Прототипы и UX-решения
- Скриншоты / wireframes главных экранов
- Логика навигации
- Почему так (1–2 UX-обоснования)

10. UI-концепция и визуальный стиль
- Цветовая палитра
- Типографика
- Общий стиль и настроение
- UI-референсы

11. Техническая часть
- Формат продукта (MVP, сервис, игра и т.д.)
- Примерная архитектура
- Технологический стек (Frontend / Backend / ML / Engine)

12. Ожидаемый результат
- Какой ценностный результат в итоге
